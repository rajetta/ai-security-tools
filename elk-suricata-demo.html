Suricata: Deep Packet Inspection

High-Performance Network Intrusion Detection System

Multi-threaded security engine with advanced threat detection capabilities

Core Capabilities

🔍 Detection Methods

  • Signature-based: Known threat patterns
  • Anomaly detection: Unusual protocol usage
  • Protocol analysis: Deep packet inspection
  • Flow monitoring: Connection tracking

📊 Logging Features

  • Structured JSON output (eve.json)
  • Detailed alert information
  • Network flow metadata
  • File extraction capabilities

Network Security Functions

🛡️ Acts Like a Security Camera

Continuously monitors network traffic performing:

  • Threat Identification: Extensive ruleset for malware, C&C activity, exploits
  • Anomaly Detection: Identifies unusual patterns indicating novel threats
  • Complete Logging: Detailed events for every alert and flow

📋 Sample Suricata Rule

# SQL Injection Detection Rule
alert http any any -> any any ( msg:"SQL Injection Attempt"; content:"union select"; nocase; http_uri; classtype:web-application-attack; sid:1001; rev:1; )

2. Shipping

Filebeat tails log file and forwards events to Logstash

3. Processing

Logstash parses, enriches (GeoIP), and structures data

↓

4. Storage & Indexing

Elasticsearch stores and indexes for instant search

↓

5. Visualization & Analysis

Kibana queries and displays real-time dashboards

🔧 Sample Logstash Configuration

# /etc/logstash/conf.d/suricata.conf
input { beats { port => 5044 } } filter { if [fields][source] == "suricata" { json { source => "message" } if [event_type] == "alert" { mutate { add_tag => ["suricata_alert"] } geoip { source => "src_ip" target => "src_geo" } } } } output { elasticsearch { hosts => ["localhost:9200"] index => "suricata-%{+YYYY.MM.dd}" } }

Live Demo - Step 1: Simulating the Attack

🎯 SQL Injection Attack Simulation

We'll simulate a common SQL Injection attack using curl to send malicious requests

💻 Attack Command

$ curl -X GET "http://vulnerable-app.local/search?q=test' UNION SELECT username,password FROM users--"
GET /search?q=test' UNION SELECT username,password FROM users-- HTTP/1.1 Host: vulnerable-app.local User-Agent: curl/7.68.0 Accept: */* Response: 200 OK Content-Type: text/html

🔍 What's Happening

  • Attack Vector: SQL Injection via URL parameter
  • Payload: UNION SELECT to extract user credentials
  • Target: Web application database
  • Detection: Suricata monitors HTTP traffic for malicious patterns

⚠️ Security Note

This demonstration uses a controlled environment. Never attempt these attacks on systems you don't own or without explicit permission.

Advanced Analysis & Threat Hunting

Beyond Basic Dashboards: Deep Investigation Capabilities

Kibana's Discover interface enables sophisticated threat hunting and forensic analysis

🔍 Interactive Threat Hunting

Explore advanced search and investigation capabilities

🔍 Kibana Query Language (KQL) Examples

# Find all SQL injection attempts in last 24 hours
alert.signature: "*SQL injection*" AND @timestamp: [now-24h TO now]
# Investigate specific attacker IP
src_ip: "192.168.1.100" AND event_type: "alert"
# Find high severity alerts
alert.severity: [1 TO 3] AND alert.category: "*attack*"
# Detect potential lateral movement
src_ip: "192.168.1.*" AND dest_ip: "10.0.1.*" AND proto: "TCP"
